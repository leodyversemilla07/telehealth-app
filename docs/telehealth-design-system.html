Color Tokens

All colors are defined in OKLCH for perceptually uniform lightness — ensuring accurate contrast ratios across themes. The palette centers on blues and teals per the design reference: "Blues and teals convey trust and calm." Avoid aggressive reds or high-saturation orange except for urgent status.

Forms & Inputs

Input height is 44px (WCAG touch target). Font size is 16px minimum. Inline validation is required — "don't make users submit to discover errors." Password fields include strength meters on registration.

Alerts & Toasts

Generic error messages are a critical UX mistake in healthcare: "Something went wrong" creates panic — always specify what failed and what to do next." Toasts are delivered via Sonner.

Accessibility — WCAG 2.1 AA

"Accessibility is required, not optional — elderly and chronically ill users are primary audiences." Design reference mandates WCAG 2.1 AA as minimum. All interactive elements must be keyboard-navigable with visible focus rings.

AI Integration Patterns

AI is an emerging expectation in telehealth UX. Key principle: "AI recommendations must show confidence levels and always provide an escape hatch — 'See all doctors / I'll choose manually.'"

Symptom → Specialty Mapper (NVIDIA NIM) ✅

AI Recommendation Confidence: 87%

Based on your symptoms (chest tightness, shortness of breath), we recommend a Cardiologist.

Always show confidence level and provide an escape hatch. AI should assist, never gate, patient choice.

Other AI Patterns

✅ Implemented Symptom-to-specialty mapper

NVIDIA NIM integration. Patients describe symptoms → AI returns specialty recommendations.

⚠ Gap Triage assistant

Helps categorize urgency before booking: non-emergency / urgent / emergency redirect.

⚠ Gap Post-consultation summary

AI generates readable consultation summary for patients from doctor's notes after COMPLETED status.

⚠ Gap Smart scheduling

AI suggests optimal appointment times based on doctor availability and patient visit history.

Performance Targets

Telehealth users in rural or low-connectivity areas must still access booking. Progressive loading and offline-friendly appointment confirmations (cached) are recommended.

MetricTargetImplication
LCP (Largest Contentful Paint)< 2.5sHero section must render fast — avoid render-blocking fonts / scripts
FID / INP (Interaction to Next Paint)< 100msButton feedback must be instant — no heavy synchronous JS on clicks
CLS (Cumulative Layout Shift)< 0.1Skeleton screens + reserved height on data-loaded card areas
TTI (Time to Interactive)< 3.5sCode-split dashboards; lazy-load video SDK until join-room
JS bundle (mobile)< 500KB initialNext.js 15 + Turbopack; tree-shake unused ShadCN components

Gap Tracker

Features identified in the design reference that are not yet fully implemented in telehealth-app. Sourced from docs/telehealth-design-reference.md § Gaps to Address.

FeaturePriorityDesign Reference NoteCodebase Status
Email notifications High No-show / no-notification is the biggest patient satisfaction failure. SMS/email reminders are expected. Currently console.log only — no SMTP transport
Messaging / secure chat High "Secure messaging with doctors" — Medium UX priority per feature table. Route exists in frontend; no backend implementation
Appointment reminder notifications High Reminder notifications are expected. NotificationType.APPOINTMENT_REMINDER exists but nothing sends it. NotificationType defined; no trigger
Doctor profile photos Medium "Doctor profile without photo = 3× lower booking conversion than profiles with photos." image field exists on User; no UI to display on doctor cards
Post-visit consultation summary Medium Consultation notes + prescriptions should be accessible post-COMPLETED. No patient-facing note view after COMPLETED status
Prescription download (PDF) Medium "Prescription history and download" — High UX priority per feature table. Stored in DB; no download endpoint
Mobile booking calendar Medium 60–70% of sessions on mobile; booking calendar needs swipe navigation. Small touch targets cause wrong slot selection. Mobile calendar UI not yet built
Multi-step booking progress indicator Low "Show progress indicators on multi-step forms" to reduce friction and abandonment. Multi-step flow has no step indicator
Pricing visible before booking Medium "Hiding pricing until booking confirmation erodes trust; patients abandon at payment step." Fee shown on doctor cards — verify it's visible before confirmation step
AI triage assistant Medium Helps categorize urgency before booking: non-emergency vs urgent vs emergency redirect. Not implemented; symptom mapper exists but no urgency tier
Post-consultation AI summary Medium "AI generates readable summary of consultation notes for patients." Not implemented
High — Blocks patient satisfaction Medium — Conversion / feature gap Low — Polish / UX improvement ✅ Done
